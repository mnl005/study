패킷 교환 방식 : 메시지를 패킷 단위로 쪼게어 메시지를 네트워크상에서 송수신, 현대 인터넷의 전송방식
패킷 스위치 : 패킷의 최적의 경로를 결정, 패킷의 송수신자 식별 (라우터, 스위치)
페이로드 : 패킷을 통해 전송하고자 하는 데이터
헤더와 트레일러 : 패킷 앞뒤로 붙는 부가정보.제어정보
프로토콜 : 정보를 주고받기 위한 합의된 규칙이나 방법

캡슐화 : 상위 계층에서 전달받은 패킷을 페이로드 삼아 프로토콜에 맞는 헤더나 트레일러를 붙여 하위 계층으로 전달
역캡슐화 : 캡슐화의 반대

대역폭 : 주어진 시간 동안 네트워크 연결 통해 흐를 수 있는 최대 정보량
트래픽 : 특정 시점에 링크 내에 흐르는 데이터 양, 데이터의 누적수
과부화 : 특정 노드에 트래픽이 몰려 해당 노드가 특정 시간동안 처리해야 할 정보가 많다
처리율 : 단위시간당 네트워크 통해 실제 전송되는 정보량,bps(bit/s),Mbps(Mbit/s), pps(ps)등의 단위 존재
패킷손실 : 전체 패킷중 유실된 패킷(ping 명령으로 백분위 단위의 패킷 손실량 확인)
지연시간 : 요청처리시간, 어떤 메시지가 두 장치 사이를 왕복시 소요되는 시간, 유선.무선인지, 패킷크기, 라우터 패킷 처리 시간에 영향 받는다

LAN : 근거리 통신망
MAN : 대도시 지역 네트워크
WAN : 광역 네트워크, ISP가 회선을 임대하는 등의 서비스를 제공

네트워크 토폴리지 : 노드와 링크의 배치 방식이자 연결형태
트리형 : 노트추가삭제원활, 특정노드에 트래픽 집중시 하위노드 영향
버스 토폴로지 : 중앙 통신회선 하나에 여러개의 노드가 연결되어 공유하는 네트워크 구성, 근거리통신망 에서 사용, 스푸핑이 가능한 문제점,설치비용적고 신뢰성우수
스푸핑 : LAN상의 송신부 패킷을 조작해 스위칭기능마비나 패킷경로를 변경
스타형 : 중앙에 있는 노드에 모두 연결된 네트워크 구성, 에러탐지용이, 패킷충돌적다,중앙노드장에시 불리
링형 : 각 노드가 양 옆의 노드에 연결, 노드수 증가시 유리, 구성변경불리, 회선장애시 영향크다
매시형 : 그물망형태, 장애발생시 복구용이, 노드추가불리,고가

유니캐스트 : 1대1 메시지 송수신
브로드캐스트 : 자신을 제외한 네트워크상 모든 호스트에게 전송
멀티캐스트 : 동일 그룹 호스트에게 전송
애니캐스트 : 동일 그룹 호스트중 가장 가까운 호스트에게 전송

토큰링 : 이더넷 이외의 LAN기술, 호스트를 링 형태로 연결해 메시지 송신시 토큰을 요구

트위스티드 페어 케이블 : 구리선으로 전기신호 주고받는다, 케이블본체와 커넥터로 구성, 전기신호 왜곡하는 노이즈 막기 위해 브레이드실드.포일실드로 차폐한다
광섬유 케이블 : 빛을 통해 정보를 주고받는다, 노이즈의 간섭 적다, 케이블본체와 커넥터로 구성, 케이블 본체는 코어와 코어를 둘러싸 빛을 가두는 클래딩으로 구다

반이중모드 : 송수신을 한쪽만 한다
전이중 모드 : 송수신을 동시에 양방향으로 한다
