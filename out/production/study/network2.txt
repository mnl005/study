패킷 교환 방식 : 메시지를 패킷 단위로 쪼게어 메시지를 네트워크상에서 송수신, 현대 인터넷의 전송방식
패킷 스위치 : 패킷의 최적의 경로를 결정, 패킷의 송수신자 식별 (라우터, 스위치)
페이로드 : 패킷을 통해 전송하고자 하는 데이터
헤더와 트레일러 : 패킷 앞뒤로 붙는 부가정보.제어정보
프로토콜 : 정보를 주고받기 위한 합의된 규칙이나 방법

캡슐화 : 상위 계층에서 전달받은 패킷을 페이로드 삼아 프로토콜에 맞는 헤더나 트레일러를 붙여 하위 계층으로 전달
역캡슐화 : 캡슐화의 반대

대역폭 : 주어진 시간 동안 네트워크 연결 통해 흐를 수 있는 최대 정보량
트래픽 : 특정 시점에 링크 내에 흐르는 데이터 양, 데이터의 누적수
과부화 : 특정 노드에 트래픽이 몰려 해당 노드가 특정 시간동안 처리해야 할 정보가 많다
처리율 : 단위시간당 네트워크 통해 실제 전송되는 정보량,bps(bit/s),Mbps(Mbit/s), pps(ps)등의 단위 존재
패킷손실 : 전체 패킷중 유실된 패킷(ping 명령으로 백분위 단위의 패킷 손실량 확인)
지연시간 : 요청처리시간, 어떤 메시지가 두 장치 사이를 왕복시 소요되는 시간, 유선.무선인지, 패킷크기, 라우터 패킷 처리 시간에 영향 받는다

LAN : 근거리 통신망
MAN : 대도시 지역 네트워크
WAN : 광역 네트워크, ISP가 회선을 임대하는 등의 서비스를 제공

네트워크 토폴리지 : 노드와 링크의 배치 방식이자 연결형태
트리형 : 노트추가삭제원활, 특정노드에 트래픽 집중시 하위노드 영향
버스 토폴로지 : 중앙 통신회선 하나에 여러개의 노드가 연결되어 공유하는 네트워크 구성, 근거리통신망 에서 사용, 스푸핑이 가능한 문제점,설치비용적고 신뢰성우수
스푸핑 : LAN상의 송신부 패킷을 조작해 스위칭기능마비나 패킷경로를 변경
스타형 : 중앙에 있는 노드에 모두 연결된 네트워크 구성, 에러탐지용이, 패킷충돌적다,중앙노드장에시 불리
링형 : 각 노드가 양 옆의 노드에 연결, 노드수 증가시 유리, 구성변경불리, 회선장애시 영향크다
매시형 : 그물망형태, 장애발생시 복구용이, 노드추가불리,고가

유니캐스트 : 1대1 메시지 송수신
브로드캐스트 : 자신을 제외한 네트워크상 모든 호스트에게 전송
멀티캐스트 : 동일 그룹 호스트에게 전송
애니캐스트 : 동일 그룹 호스트중 가장 가까운 호스트에게 전송

토큰링 : 이더넷 이외의 LAN기술, 호스트를 링 형태로 연결해 메시지 송신시 토큰을 요구

트위스티드 페어 케이블 : 구리선으로 전기신호 주고받는다, 케이블본체와 커넥터로 구성, 전기신호 왜곡하는 노이즈 막기 위해 브레이드실드.포일실드로 차폐한다
광섬유 케이블 : 빛을 통해 정보를 주고받는다, 노이즈의 간섭 적다, 케이블본체와 커넥터로 구성, 케이블 본체는 코어와 코어를 둘러싸 빛을 가두는 클래딩으로 구다

반이중모드 : 송수신을 한쪽만 한다
전이중 모드 : 송수신을 동시에 양방향으로 한다

라우팅 : 패킷이 이동할 최적의 경로를 결정

경로 MTU : IP단편화 없이 주고 받을수 있는 최대의 크기
경로 MTU 발견 : 경로 MTU를 구하고 해당 크기만큼 송수신하여 단편화를 회피하는 기술, 이를 통해 IP단편화는 자주 일어나지 않는다

게이트웨이 : 호스트가 속한 네트워크의 외부로 나가기 위한 첫 경로를 기본 게이트웨이, 즉 라우터(공유기)
루프백 주소 : 172.0.0.0/8 - 로컬호스트 주소이다

잘 알려진 포트 : 0 ~ 1023
등록된 포트 : 1024 ~ 49151
동적포트 : 49152 ~ 65535
FTP : 20,23
SSH : 22
TELNET : 23
DNS : 53
DHCP : 67,68
HTTP : 80
HTTPS : 433
+ 포트 번호의 예시는 권고사항이다
192.168.0.15:8000
+ ip주소로 호스트 식별하고 포트번호로 애플리케이션 프로세스를 식별한다

HTTP 메서드 :
... GET : 자원습득위한 메서드
... HEAD : GET과 동일하나 헤더만을 응답받는 메서드
... POST : 서버로 하여금 특정 작업 처리하게 하는 메서드
... PUT : 자원을 대체하기위한 메서드
... PATCH : 자원에 대한 부수적 수정을 위한 메서드
... DELETE : 자원을 삭제하기 위한 메서드
... CONNECT : 자원에 대한 양방향 연결을 시작하는 메서드
... OPTIONS : 사용 가능한 메서드 등 통신 옵션을 확인하는 메서드
... TRACE : 자원에 대한 루프백 테스트 수행하는 메서드

HTTP 상태코드 :
100번대 : 정보성 상태 코드
200번대 : 성공상태코드
300번대 : 리다이렉션 상태코드
400번대 : 클라이언트 에러 상태코드
500번대 : 서버 에러 상태 코드

리다이렉션 : 요청을 완수하기 위해 추가적인 조치가 필요한 상태

인증 : 자신이 누구인지 증명하는것
권한부여 : 인증된 주체에게 작업을 허용하는것

안전성 : 특정 기능을 언제든 균일한 성능으로 수행할 수 있는 특성
가용성 : 컴퓨터 시스템이 특정 기능을 실제로 수행 할 수 았는 시간의 비율
결함감내 : 문제가 발생하더라도 기능할 수 있는 능력
이중화 : 결함을 감내하여 가용성을 높이기 위한 기본적인 방법
다중화 : 이중화의 확장개념, 다중화 통해 안정적 운영을 추구
티밍/본딩 : 윈도우/리눅스 에서 여러개의 네트워크 인터페이스(NIC)를 이중화.다중화 하는것
로드 밸런싱 : 트래픽의 고른 분배를 위해 사용
로드 밸런서 : L4나 L7 스위치로 구성하거나 로드 밸런싱 기능 제공하는 서버를 사용
헬스 체크 : 다중화된 서버 환경에서 서버의 문제점 감지

라운드로빈알고리즘 : 서버를 돌아가며 부하전달
최소 연결 알고리즘 : 연결이 적은 서버부터 우선적으로 부하를 전달
