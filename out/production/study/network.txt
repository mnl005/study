
네트워크 : 노드와 링크가 연결되었거나 연결된 상태에서 리소스 공유하는 집합
노드 : 서버,라우터,스위치 등의 네트워크 장치
링크 : 유선 또는 무선의 연결

처리량 : 링크 내에 전달완료 된 데이터의 양, 트래픽 처리량,
... 단위는 bps(bits per second), 트래픽.네트워크장치간 대역폭.네트워크종간에러.하드웨어 스펙에 영향
대역폭 : 주어진 시간 동안 네트워크 연결 통해 흐를 수 있는 최대 비트수
트래픽 : 특정 시점에 링크 내에 흐르는 데이터 양, 데이터의 누적수

지연시간 : 요청처리시간, 어떤 메시지가 두 장치 사이를 왕복시 소요되는 시간, 유선.무선인지, 패킷크기, 라우터 패킷 처리 시간에 영향 받는다

네트워크 토폴리지 : 노드와 링크의 배치 방식이자 연결형태
트리형 : 노트추가삭제원활, 특정노드에 트래픽 집중시 하위노드 영향
버스 토폴로지 : 중앙 통신회선 하나에 여러개의 노드가 연결되어 공유하는 네트워크 구성, 근거리통신망 에서 사용, 스푸핑이 가능한 문제점,설치비용적고 신뢰성우수
스푸핑 : LAN상의 송신부 패킷을 조작해 스위칭기능마비나 패킷경로를 변경
스타형 : 중앙에 있는 노드에 모두 연결된 네트워크 구성, 에러탐지용이, 패킷충돌적다,중앙노드장에시 불리
링형 : 각 노드가 양 옆의 노드에 연결, 노드수 증가시 유리, 구성변경불리, 회선장애시 영향크다
매시형 : 그물망형태, 장애발생시 복구용이, 노드추가불리,고가

LAN : 근거리 통신망
MAN : 대도시 지역 네트워크
WAN : 광역 네트워크


병목현상 : 전체 시스템의 성는 또는 용량이 하나의 구성요소에 의해 제한되는 것
... 원인은 네트워크데역폭, 네트워크 토폴리지, 서버cpu, 메모리 사용량, 비효율적 네트워크 구성

ping : 네트워크 상태 확인시 대상 노드에 일정 크기의 패킷을 전송, 패킷수신상태와 도달시간 확인, 연결상태 확인, TCP/IP 나 ICMP 통해 동작
ex) ping www.google.com -n 12 : -n 12 옵션을 넣어 12번 패킷 보내고 12번 받는다

netstat : 접속된 서비스의 네트워크 상태표시, 네트워크접속,라우팅테이블,네트워크 토폴리지등의 리스트 확인, 서비스 포트 열림 유무 확인
ex) netstat

nslookp : DNS관련 내용 확인, 특정 도메인에 매핑된 IP확인
ex) nslookup

tracert : 목적지까지 네트워크 경로를 확인, 어느 구간에서 응답이 느려지는지 확인
ex) tracert www.google.com

네트워크 프로토콜 표준화 : 서로다른 장치간 데이터 교류 위해 설정된 공통된 인터페이스
ex) 웹은 HTTP

네트워크 계층 모델 : 네트워크에서 사용되는 통신 프로토콜의 집합, 각 계층은 프로토콜의 네트워킹 범위따라 추상화 계층으로 분류
... 특정 계층 변경시 다른 계층에 영향을 주지 않는다
OSI 7계층 : 물리,데이터링크,네트워크,전송,세션,프레젠테이션,애플리캐이션
TCP/IP 4계층 : 링크,인터넷,전송,애플리캐이션 계층
... 링크계층 : 이더넷
... 인터넷계층 : IP,ARP,ICMP
... 전송계층 : TCP,UDP,QUIC
... 애플리케이션 계층 : FTP,HTTP,SSH,SMTP,DNS

FTP : 장치와 장치간 파일 전송시 사용되는 표준 프로토콜
SSH : 보안되지 않은 네트워크에서 서비스를 안전하게 운영하기 위한 암호화된 프로토콜
HTTP : www를 위한 데이터 통시느이 기초, 웹사이트 이용시 사용
SMTP : 전자 메일 전송 위한 인터넷 표준 프로토콜
DNS : 도메인 이름과 IP주소 매핑

전송계층 : 송수신자간 연결 서비스 제공, 연결 지향 데이터 스트림 지원,신뢰성, 흐름제어
... 애플리캐이션과 인터넷계층 사이의 데이터 중계역할 하는 TCP, UDP
UDP : 순서보장없음, 수신여부 확인없음, 데이터그램 패킷 교환 방식

TCP : 패킷순서보장, 연결지향으로 수신여부 확인, 가상회선 패킷 교환 방식
... 신뢰성 위해 3웨이핸드셰이크 작업 진행
... SYN 단계 : 클라이언트가 서버에 ISN담아 SYN 전송, ISN은 첫 tcp 연결의 첫번째 패킷에 할당된 임의의 시퀸스 번호
... SYN + ACK 단계 : 서버는 클라이언트의 SYN을 수신하고 서버의 ISN을 보내고 승인번호로 클라이언트의 ISN+1을 전송
... ACK 단계 : 클라이언트는 서버의 ISN+1한 값인 승인번호를 담아 ACK를 서버에 전송
.....SYN(연결요청플래그), ACK(응답플래그), ISN(초기네트워크 연결시 할당된 32비트 고유 시퀸스 번호)

... 연결해제시 4웨이 핸드셰이크 진행
... 1 : 클라이언트가 연결 닫을시 FIN으로 설정된 세그먼트 전송, 클라이언트는 FIN_WAIT_1 상태로 서버의 응답 기다림
... 2 : 서버는 클라이언트로 ACK승인 세그먼트 전송 CLOSE_WAIT 상태로 있다가 클라이언트가 세그먼트 받으면 FIN_WAIT_2 상태로 들어감
... 3 : 서버는 ACK를 보낸후 일정 시간 이후 클라이언트에 FIN 세그먼트 전송
... 4 : 클라이언트는 TIME_WAIT상태가 되고 다시 서버로 ACK를 보내 서버는 CLOSED상태가 됨, 이후 클라이언트는 일정시간 대기후 연결닫히고 자원연결해제
..... TIME_WAIT의 이유는 지연패킷을 고려하기 위함

인터넷 계층 : 장치부터 받은 네트워크 패킷을 IP주소로 지정된 목적지로 전송하기 위한 계층, 패킷 수신할 상대 주소 지정해 데이터 전달
링크 계층 : 전선,광섬유 등으로 실질적으로 데이터를 전달, 장치간 신호 주고받는 규칙을 정한다
... 유무선 랜으로 이진데이터 전송하는 물리계층과 이더넷 프레임 통해 에러확인.흐름제어.접근제어 담당하는 데이터 링크 계층으로 나뉨
유선랜 : 양쪽 장치가 동시에 송수신 가능한 전이중화 방식을 사용, 현대의 고속 이더넷이 이 경우에 해당
... 반이중화 통신 : 데이터 전송후 충돌 발생시 일정시간 이후 재전송
... 트위스트 페어 케이블(구리선), UTP케이블(실드처리없는 구리선), STP케이블(실드처리 후 덮은 캐이블), 광섬유 케이블

CSMA/CA : 반이중화 통식 방식, 데이터 보내기 이전 일련의 과정으로 사전 충돌 방지
... 유후상태채널발견 -> 프레임 간 공간 시간(IFS)시간 만큼 대기, IFS가 낮으면 우선순위 높다 -> 프레임 전송이전
..... 랜덤상수만큼 기다린 후 프레임 전송 -> 전송후 ACK세그먼트 받으면 마침, 받지못했다면 kmax만큼 반복하다 안되면 버림
... 전이중화 통신은 양방향 통신이 가능해 충돌 가능성이 없어 충돌감지나 방지 매커니즘이 필요없다

무선랜 : 무선신호 전달로 2대 이상의 장치를 연결
... 2.4GHz대역은 장애물에 강하나 전파간섭이 많다, 5GHz는 사용가능한 채널수가 많도 동시사용 가능하다
와이파이 : 유선LAN의 신호를 무선 LAN신호로 바꿔 신호 닿는 범위내에서 무선 인터넷 사용 가능
BSS : 기본서비스의 집합, 영역 내의 장치끼리 통신 가능한 구조, 이동하며 네트워크 접속은 불가
ESS : 하나 이상의 연결된 BSS그룹, 장거리 무선통신, 이동하며 네트워크접속 가능

이더넷 프레임 : 전달받은 데이터의 에러 검출과 캡슐화
... Preamble | SFD | DMAC | SMAC | EtherType | Payload | CRC
Preamble : 이더넷 프레임이 시작임을 알림
SFD : 다음 바이트부터 MAC 주소 필드가 시작됨을 알림
DMAC,SMAC : 수신,송신 MAC 주소
EtherType : 데이터 계층 위의 계층인 IP프로토콜을 정의
Payload : 전달받은 데이터
CRC : 에러 확인 비트

MAC주소 : 컴퓨터의 각 장치의 네트워크 연결위한 6바이트의 식별번호

계층간 데이터 송수신 과정 :
.. 애플리케이션 계층에서 전송 계층으로 보내는 요청들이 캡슐화 과정을 거쳐 전달되고, 링크계층을 통해 해당 서버와 통신후
.. 해당 서버의 링크 계층으로부터 에플리케이션까지 비캡슐화 과정을 거쳐 데이터 전송

... 캡슐화 : 상위 게층의 해더와 데이터를 하위 계층의 데이터 부분에 포함시켜 해당 계층의 헤더를 삽입하는과정
..... 애플리케이션 계층의 데이터가 전송계층으로 전달되면서 세그먼트 또는 데이터그램화 되고 TCP 헤더가 붙여짐
..... 이후 인터넷 계층으로 가면서 IP 헤더가 붙여지게 되고 패킷화가 되고
..... 이후 링크계층으로 전달되면서 프레임 헤더와 프레임 트레일러가 붙어 프레임화가 된다

... 비캡슐화 : 캡슐화된 데이터를 받아 링크계층에서부터 타고 올라오며 프레임화된 데이터는 다시 패킷화를 거쳐 세그먼드, 데이터그램 화를
... 거쳐 메시지화 되는 비캡슐화 과정이 일어나고 최종적으로 사용자에게 애플리케이션의 PDU인 메시지로 전달된다

PDU : 네트워크의 어떠한 계층에서 계층으로 데이터 전달시 한 덩어리의 단위를 지칭, 각 계층별 부르는 명칭이 다르다
... 애플리케이션의 메시지, 전송계층의 세그먼트(TCP) 와 데이터그램(UDP), 인터넷계층의 패킷, 링크계층의 프레임과 비트

네트워크 기기 처리 범위 -
애플리케이션 계층 : L7 스위치
인터넷 계층 : 라우터, L3스위치
데이터 링크 계층 : L2스위치, 브리지
물리계층 : NIC, 리피터, AP

L7 스위치 : 스위치는 여러 장비 연결과 데이터 통신 중재하고 목적지가 연결된 포트로만 전기신호 보내 데이터 전송
...L7스위치는 로드밸런서라고도 불린다, 서버의 부하 분산, 클라이언트들의 요청들을 여러 서버로 나누어 시스템의 처리가능한 트래픽을 증가시킨다
... URL, 서버, 캐시, 쿠키들을 기반으로 트래픽을 분산하고 필터링과 트래픽 모니터링 가능, 장애발생 서버를 헬스체크 이용하여 트래픽 분산 대상에서 제외
... L4 스위치는 전송계층을 처리하며 스트리밍 기반 서비스에서 사용 불가, 메시지 기반의 인식 불가, IP와 포트 기반으로 트래픽을 분산

헬스체크 : 전송주기와 재전송 횟수 등을 설정해 반복적으로 서버에 요청을 보내는 것
로드밸런서 : 2대 이상의 서버를 기반으로 가상 IP제공해 안정적 서비스를 제공

라우터 : 여러개의 네트워크를 연결,분할,구분 시켜주고 다른 네트워크에 존재하는 장치끼리 서로 데이터 주고받을 때 패킷 소모를 최소화하고 경로를 최적화하여
... 최소 경로로 패킷을 포워딩 하는 라우팅 장비

L3 스위치 : 하드웨어 기반의 라우팅과 소프트웨어 기반의 라우팅중 하드웨어 기반의 라우팅을 담당하는 장치
L2 스위치 : MAC주소를 MAC주소 테이블 통해 관리하고 연결된 장치로부터 패킷이 왔을 때 패킷전송을 담당

브리지 : 두개의 근거리 통신망을  상호 접속할 수 있도록 하는 통신망 연결 장치, 포트와 포트 사이의 다리역할, 장치에서 받아온 MAC주소를 MAC주소 테이블로관리
... 브리지는 통신망 범위를 확장하고 서로다른 LAN등으로 이루어진 하나의 통신망을 구축할 때 쓰임

물리계층 처리 기기 -
NIC : LAN카드라 하는 네트워크 인터페이스 카드는 2대 이상의 컴퓨터 네트워크를 구성시 사용, 네트워크와 빠른 속도로 데이터를 송수신 할 수 있도록
... 컴퓨터 내 설치하는 확장카드, 각 LAN카드는 각각을 구분하기 위한 고유 식별 번호인 MAC주소가 존재
리피터 : 약해진 신호 정도를 증폭해서 다른 쪽으로 전달하는 장치, 패킷이 더 멀리갈 수 있으나 광케이블의 보급으로 현재 잘 안 쓰임
AP : 패킷 복사 장치

ARP : IP 주소로부터 MAC주소 구하는 장치, 반대는 RARP
브로드캐스트 : 송신 호스트가 전송한 데이터가 네트워크에 연결된 모든 호스트에 전송하는 방식
유니캐스트 : 고유 주소로 식별된 하나의 네트워크 목적지에 1:1로 데이터를 전송하는 방식

홉바이홉 통신 : IP주소 통해 통신하는 과정, 통신 장치의 라우팅 테이블의 IP를 통해 시작 주소부터 다음 IP로 이동하는 라우팅 과정 거쳐
... 패킷이 최종 목적지까지 도달하는 통신
라우팅 : IP주소를 찾아가는 과정
라우팅 테이블 : 송신지에서 수신지까지 도달하기 위해 사용, 라우터에 들어가 있는 목적지 정보들과 그 목적지로 가기위한 방법이 있는 리스트,
... 게이트웨이와 모든 목적지에 대해 해당 목적지에 도달하기 위해 거쳐야 할 다음 라우터의 정보를 갖추고 있다
게이트웨이 : 서로 다른 통신망,프로토콜을 사용하는 네트워크 간 통신을 가능하게 하는 관문 역할을 하는 컴퓨터나 소프트웨어의 총칭

IPv4 : 32비트를 8비트 단위로 점찍어 표현, 클래스 기반 할당 방식
IPv6 : 64비트를 16비트 단위로 점찍어 표현

DHCP : IP주소 및 기타 통신 매개변수를 자동으로 할당하기위한 네트워크 관리 프로토콜, 인터넷 접속시 자동으로 IP주소 할당 가능
NAT : 패킷이 라우팅 장치를 통해 전송되는 동안 패킷의 IP주소 정보를 수정하여 IP주소를 다른 주소로 매핑
... IPv4만으로 처리못할 많은 주소를 처리 가능, 여러대의 호스트가 하나의 공인 IP주소 사용해 인터넷에 접속, 보안시 유리, 접속 속도가 저하 가능

HTTP/1.0 : 한 연결당 하나의 요청 처리, 서버로부터 파일 가져올 때마다 TCP 3웨이 핸드셰이크를 열어야 함으로 RTT가 증가하는 단점
RTT : 패킷이 목적지에 도달후 다시 출발지로 돌아오기까지 걸리는시간, 패킷 왕복 시간
... 이미지 스플리팅 : 많은 이미지가 합쳐진 하나의 이미지를 다운받고 이를 기반으로 css의 배경으로 이미지를 표기
... 코드압축 : 개행문자,빈칸을 없에 코드 크기를 최소화
... 이미지 Base64 인코딩 : 이미지 파일을 64진법으로 이루어진 문자열로 인코딩
인코딩 : 정보의 형태나 형식을 표준화,보안,처리속도향상,저장공간절약 등 위해 다른 형태나 형식으로 변환하는 처리방식

HTTP/1.1 : 한번 TCP초기화 이후 keep-alive 옵션으로 여러개의 파일의 송수신 가능하게 됨
HOL Blocking : 네트워크에서 같은 큐이 있는 패킷이 그 첫번째 패킷에 의해 지연될 때 발생하는 성능 저하 현상

HTTP/2 : 1.x 버전보다 지연시간 줄이고 응답시간 빠르게해서 멀티플렉싱,헤더압축,서버푸시,요청의 우선처리를 지원하는 프로토콜
멀티플렉싱 : 여러개의 스트림 이용한 송수신, 특정스트림의 패킷손실이 되더라도 나머지 스트림은 정상작동 가능
스트림 : 시간이 지남에 따라 사용할 수 있게되는 일련의 데이터 요소를 가르키는 데이터 흐름
헤더압축 : 허프만 코딩 압축 알고리즘을 사용하는 HPACK 압축 형식을 가진다
허프만코딩 : 문자열을 문자 단위로 쪼개 빈도수를 세어 빈도가 높은 적보는 적은 비트수를 사용하고 빈도가 낮은 정보는 비트수를 많이 사용해 전체
... 데이터 표현에 필요한 비트양을 줄인다
서버푸시 : 클라이언트 요청 없이 서버가 바로 리소스를 푸시한다, html을 읽으면 서버에서 css나 js파일을 전송한다

HTTPS : HTTP/2는 HTTPS위에서 동작한다, 애플리케이션 계층과 전송계층 사이의 신뢰계층인 SSL/TLS계층을 넣은 신뢰 가능한 HTTP요청을 의미, 통신을 암호화

SSL/TLS : 전송 계층의 보안 제공 프로토콜, 클라이언트와 서버의 통신시 제3자가 메세지를 도청 또는 변조를 막는다, 인터셉터를 방지,
... 보안세션 기반으로 데이터를 암호화하고 보안 세션이 만들어질 때 인증 매커니즘,키교환암호화 알고리즘, 해싱 알고리즘이 사용
보안세션 : 보안이 시작되고 끝나는 동안 유지되는 세션, SSL/TLS는 핸드셰이크 통해 보안 세션 생성하고 이를 기반으로 상태 정보 등을 공유
세션 : 운영체제가 어떠한 사용자로부터 자신의 자산 이용을 허락하는 일정한 기간을 의미, 사용자는 일정 시간동안 응용 프로그램 자원 등을 사용 가능,
... 클라이언트에서 사이퍼 슈트를 서버에 전달하면 서버는 받은 사이퍼 슈트의 암호화 알고리즘 리스트를 제공가능한지 확인 -> 제공 가능하다면
... 서버에서 클라이언트로 인증서를 보내는 인증 메커니즘이 시작 -> 해싱 알고리즘 등으로 암호화된 데이터의 송수신이 시작
사이퍼 슈트 : 프로토콜, AEAD사이퍼모드, 해싱 알고리즘이 나열된 규약 총 5개
AEAD 사이퍼 모드 : 데이터 암호화 알고리즘, AES_128_GCM은 128비트의 키를 사용하는 표준 블록 암호화 기술과 병렬 계산에 용이한 암호화 알고리즘
... GCM이 결합된 알고리즘을 의미
인증 메커니즘 : CA에서 발급한 인증서를 기반으로 이루어짐, 인증서는 안전한 연결을 시작하는데 있어 필요한 공개키를 클라이언트에 제공하고 사용자가 접속한 서버가
... 신뢰할 수 있는 서버임을 보장, 인증서 서비스는 정보.공개키.지문.디지털서명 등으로 이루어짐, 신뢰성 높은 공인된 기업만 참여가능
암호화 알고리즘 : 키교환 암호화 알고리즘으로 대수곡선 기반의 ECDHE 또는 모듈식 기반의 DHE를 사용하며 둘 다 디피-헬만 방식으로 만들어짐
디피-헬만 키 교환 암호화 알고리즘 : 공개값을 공유후 각자의 비밀값과 혼합후 혼합값을 공유 -> 공유받은 혼합값에 각자의 비밀값을 한번 더 혼합 -> PSK생성
... 이 과정으로 악의적 공격자가 개인키 또는 공개키를 가지고도 PSK가 없다면 공격불가

해싱 알고리즘 : 데이터 추정이 함든 더 작고 섞여있는 조각으로 만드는 알고리즘, SSL/TLS는 행싱 알고리즘으로 SHA-256과 SHA-384를 사용
SHA-256 : 해시 함수의 결과값이 256비트인 알고리즘이고 비트 코인을 비롯한 블록체인 시스템에서도 사용, 해싱 해야할 메시지에 1을 추가하는등 전처리후 해시반환
해시 : 다양한 길이를 가진 데이터를 고정된 길이를 가진 데이터로 매핑한 값
해싱 : 임의의 데이터를 해시로 바꿔준다, 해시 함수가 이를 담당
해싱함수 : 임의의 데이터를 입력으로 받아 일정한 길이의 데이터로 바꿔주는 함수

SEO : 검색엔진 최적화, 사이트 link에 케노니컬설정, meta를 잘 설정, 페이지 속도계선
HTTPS구축 방법 : CA에서 구매한 인증키 기반의 구축, 서버 앞단에 HTTPS를 제공하는 로드벨런스를 둔다, 서버 앞단에 HTTPS를 제공하는 CDN을 둔다 등

HTTP/3 : QUIC계층 위에서 작동, UDP 기반의 작동, 멀티플렉싱, 초기연결설정시 지연시간 감소, TCP가 아니기 때문에 3웨이헨드셰이크 불필요
QUIC : 순방향 오류 수정 매커니즘의 적용으로 전송패킷이 손실되도 수신측에서 에러 검출하고 수정가능, 열악한 네트워크 환경에서도 낮은 패킷 손실
